## 对象池模式
通过一组初始化的对象构成一个对象池。当客户端需要使用对象池中的对象时，可以将他们借出。当使用结束之后，不去销毁对象，而是归还给对象池。

#### 使用场景
* 当对象初始化成本很高并且对象可以反复使用时

#### 优点
* 通过借用、归还实现了对象的重复利用，可以节省大量的资源、时间，提升程序性能。

#### 缺点
* 借用和归还必须成对出现，不然这个对象一直处于被占用状态。
* 对已归还的对象，不能再进行任何其他操作。否则容易产生不可预料的结果。

#### 场景案例
* 各种连接池
